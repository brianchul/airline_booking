# 題目

設計特定航空的訂機票的功能，規格：

1. 可以按照起點、目的地、日期查詢班機狀態
2. 採取分頁方式返回可用班機清單、價格以及剩餘機位
3. 航空公司有超賣的慣例，功能也需要考量到超賣的情境
4. 設計表結構和索引、編寫主要程式碼、考慮大流量高並發情況(可以使用虛擬碼實現)。

## 🎯 關鍵技術選型建議

### 後端技術棧

- **框架**: Spring Boot 3.0+ / Go Gin / Node.js Express
- **資料庫**: MySQL 8.0 (主) + Redis (快取)
- **訊息佇列**: RabbitMQ / Apache Kafka
- **搜尋引擎**: Elasticsearch (航班搜尋優化)

### 架構模式

- **微服務架構**: 航班服務、訂票服務、支付服務、用戶服務分離
- **CQRS 模式**: 讀寫分離，查詢優化
- **事件驅動**: 庫存變更、訂票狀態變更事件

### 性能優化策略

- **資料庫分片**: 按航空公司 + 時間維度分片
- **讀寫分離**: 查詢走從庫，寫入走主庫
- **快取策略**: 多層快取 + 智能預熱
- **CDN 加速**: 靜態資源與 API 響應快取

## 📈 預期性能指標

- **查詢接口**: 響應時間 < 200ms，QPS > 10000
- **訂票接口**: 響應時間 < 500ms，QPS > 5000
- **系統可用性**: 99.95%以上
- **庫存準確性**: 99.9%以上
- **超賣率控制**: < 2%

## ⚠️ 風險點與應對措施

1. **庫存超賣風險**: 實現分散式鎖 + 樂觀鎖雙重保護
2. **高並發性能風險**: 多層快取 + 限流熔斷
3. **數據一致性風險**: 分散式事務 + 補償機制
4. **系統可用性風險**: 多活部署 + 故障轉移
5. **安全性風險**: 多層安全防護 + 審計監控
